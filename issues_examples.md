# Exemplos de Issues e Labels para Whack-a-Mole

## Rótulos sugeridos

- **enhancement** – novas funcionalidades ou melhorias
- **bug** – correção de comportamentos inesperados
- **documentation** – alterações na documentação
- **help wanted** – tarefas abertas para contribuições externas
- **good first issue** – tarefas simples para novos contribuidores

## Issues planejadas

1. **Adicionar multiplayer local**
   - *label*: enhancement
   - Descrição: permitir dois jogadores alternarem turnos e comparar pontuações.

2. **Salvar histórico de pontuações em arquivo JSON**
   - *label*: enhancement
   - Descrição: além do `localStorage`, registrar partidas em um arquivo `scores.json` do servidor.

3. **Corrigir bug no cronômetro que não reseta em reinícios rápidos**
   - *label*: bug
   - Descrição: se o jogador iniciar um novo jogo imediatamente após o fim, a contagem pode falhar.

4. **Adicionar README com instruções de deploy**
   - *label*: documentation
   - Descrição: explicar como publicar no GitHub Pages ou similar.

5. **Aprimorar estilos para dispositivos móveis**
   - *label*: enhancement, good first issue
   - Descrição: tornar o layout responsivo e tocar as toupeiras por toque.

Cada issue seria criada no GitHub usando a interface web ou via API/CLI. A lista acima é um exemplo de planejamento que poderia existir no repositório.
